## 2. 领域全景：2025–2026 各大实验室/顶会在做什么

### 2.1 VLA 基础模型（最热、竞争最激烈）
- **Physical Intelligence（π 系列）**：π0 用流匹配（flow matching）生成连续动作；π0.5 两级推理（先推理语义子任务再出动作），主打开放世界移动操作。团队 200 人、估值 110 亿美元——大厂主场。
- **OpenVLA**、**GraspVLA**（银河通用+智源+港大，端到端抓取基础模型）、**WholebodyVLA**（ICLR 2026，全身移动操作统一 VLA）。
- 学术小团队的玩法不是从头训 VLA，而是**改进它**：SimpleVLA-RL（清华+上海AI Lab，RL 后训练把真机成功率 17.5%→38.5%）、VOTE（轨迹集成投票）、AffordVLA（affordance 注入）。创新点集中在：动作表示、推理结构、用 RL/少量数据修正 VLA。

### 2.2 数据采集与数据引擎（学术界性价比最高的赛道之一）
- **UMI**（斯坦福/哥大，手持夹爪脱离机器人采数据）→ **FastUMI**（上海AI Lab，硬件解耦）→ UMI-3D、EgoGuide、人类第一视角数据（PhysBrain、EgoLive）。
- 核心逻辑：机器人数据太贵，**谁能便宜地造数据谁就有贡献**。

### 2.3 触觉与接触丰富操作（contact-rich）
- ICRA 2025 最佳论文之一 **PolyTouch**（触觉传感器+触觉扩散策略）；RSS 2025 **Reactive Diffusion Policy**（视觉-触觉快慢双层策略）；**TacDiffusion**（力域扩散策略）。
- **HIL-SERL**（Science Robotics 2025，人在环 RL 做精密装配）。
- **RSS 2026 开了 "Lab to Production" workshop**：主题即亚毫米装配、工业感知、失败检测与恢复——学界公认"不够解决、工业急需"。

### 2.4 工业 bin picking / 6D 位姿（和本项目最贴近）
- **2025 OpenCV Bin-Picking Perception Challenge**（450+ 队）总结论文的两个反直觉结论：
  ① 在目标物体数据上专门训练的方法反而不如 zero-shot 方法；② 指标选择严重影响排名。
  工业件（无纹理、反光、紧密堆叠）**仍远未解决**。
- 获胜方案 **Pickalo**：低成本硬件（**UR5e + RealSense，和本项目几乎同配置**）+ 腕装相机主动多视角 + SAM-6D，重遮挡 90% 抓取成功率，发表于 2026。**证明本项目量级的硬件足够发论文。**
- 其他：SDT-6D（多视角纯深度工业位姿）、位姿**分布**估计（显式建模旋转对称/歧义不确定性）。

### 2.5 灵巧手与移动操作
- 灵巧手：ICRA 2025 最佳操作论文 D(R,O) Grasp（跨手型统一抓取表征）、Dexonomy——需灵巧手硬件。
- 移动操作：OK-Robot、π0.5、MoMaGen（自动生成移动操作演示数据）——对应本项目方向 G，但系统工程量大、竞争者多。

### 3.4 规律总结
大公司/大 lab 做"通用大模型+海量数据"；学术组的创新点集中在四类：
① 新的数据获取方式（UMI 系）；② 给现成大模型打补丁（RL 后训练、不确定性、affordance）；
③ 没解决的物理 niche（工业件、透明物、紧密堆叠、亚毫米装配）；④ 新传感模态（触觉）。
**本项目条件最适合 ③，兼顾 ②。**

## 4. 论文选题方案（结合本项目硬件：UR5 + Gemini2 + 4090 + 平行夹爪）

策略：避开 VLA 大模型/灵巧手/海量数据的正面战场，**利用已搭好的完整真机系统，攻大 lab 看不上、但工业界公认没解决的 niche**。

### 选题 1（主推）：对称工业零件的"不确定性感知抓取 → 主动在手消歧 → 精密定向放置"闭环

- **任务**：从料框抓取无序堆放的对称/近对称 3D 打印零件，亚毫米/亚度级定向放置（= 项目原始目标学术化）。
- **三个创新点（论文 claim）**：
  1. **位姿分布代替单点估计**：现有方法输出单一位姿，对称零件旋转歧义被随机坍缩。改为输出多假设位姿分布（PPF/ICP 天然产生多候选，保留+打分即可），显式建模对称群。
  2. **抓取选择以"放置不确定性"为目标**：不选"最好抓的"，选"抓后歧义最易消除、放置误差最小"的抓取——与所有现有抓取规划的差异点。
  3. **单臂主动在手消歧**：抓起后规划"展示动作"，把最能区分歧义的面转向相机重估计（信息增益最大视角）。对比 RA-L 2025 双臂 regrasp，卖点是**单臂+零额外硬件**。
- **复用**：阶段 2 感知、阶段 4 inhand_estimator、pick_loop、放置补偿 —— 约 70% 已有。
- **新增**：多假设位姿输出+对称性分析（改 `pose_estimation.py`）、视角选择规划器（可简单网格搜索信息增益）、放置精度测量（ArUco 基准底座或相机复检）。
- **实验设计**：消融 = 单点 vs 分布 × 固定视角 vs 主动视角 × 有无补偿；指标 = 放置误差(mm/°)、成功率、周期时间；基线 = FoundationPose 单点+直接放置；零件 = 自打印 3–5 种不同对称性（圆柱/方块/L形）。
- **投稿**：RA-L（首选）/ ICRA / IROS / CASE；可先投 RSS L2P 类工业 workshop 拿反馈。
- **写法**：算法+系统论文（核心 claim 是不确定性感知的抓取-放置决策，系统是载体）。

### 选题 2：经典管线做教师的真机自动数据引擎（无人工遥操作的自我改进拣选）

- **任务**：PPF+ICP+MoveIt 管线（准但慢）当"教师"，真机无人值守自动采集抓取数据（成功/失败标签由夹爪开度+位姿复检自动打），蒸馏快的学生策略（ACT / HGGD 微调 / 小 VLA LoRA，4090 够）。
- **创新点**：
  1. **零人工示范的真机数据引擎**（UMI/ALOHA 都要人示范；本方案机器人自己造数据，抓了倒回料框无限循环）。
  2. **失败数据免费获得**：自动带标签，可训练失败预测/恢复（BPC 点名的 open problem）。
  3. 量化：学生比教师快 N 倍、数据量-成功率曲线、新零件泛化。
- **复用**：pick_loop、感知、夹爪驱动（约 60%）。**新增**：自动标签判定、数据记录（用 LeRobot 格式，为方向 F 铺路）、训练脚本、无人值守安全逻辑。
- **投稿**：RA-L / IROS / CASE。写法：系统/数据论文。

### 选题 3（备选/加分模块）：紧密堆叠小件的推-抓协同与失败恢复

- 基于 `model_free_grasp`：检测无夹爪插入空间 → 指尖推拨（singulation）→ 再抓；配失败检测与自动重试。
- 单独成篇偏轻，更适合作为选题 1 的鲁棒性模块（多一个消融项）或 workshop 短文。

### 执行路线
1. **先真机跑通阶段一**（标定、参数）——所有选题的地基，唯一硬门槛。
2. 主线做选题 1（复用度最高、创新最清晰、消融现成）。
3. 跑选题 1 实验时顺手搭选题 2 的数据记录——**两者共用同一 pick 循环，一套真机时间产出两篇素材**。
4. 选题 3 视时间作为加分模块。

## 5. 具身智能任务分类

### 5.1 操作类（Manipulation）—— 本项目所在

| 任务 | 说明 | 代表工作 |
|---|---|---|

| 长时程操作 | 收拾桌面、做饭（任务分解） | SayCan、π0.5、VoxPoser |

### 5.2 导航类
点导航/物体导航（Habitat）、视觉语言导航 VLN（R2R、VLN-CE）、开放词汇寻物（CLIP-Fields、VLMaps）。

### 5.3 移动操作（= 方向 G）
导航+操作合体：开放词汇取放（OK-Robot）、全屋整理（π0.5）、全身协同（WholebodyVLA）。被认为是具身智能落地家庭/工厂的关键形态。

### 5.4 人形/腿足
locomotion、loco-manipulation、人类动作模仿、全身控制——需人形/四足硬件，与当前设备不搭。

### 5.5 具身推理与规划
任务规划（SayCan、Code as Policies）、affordance/空间推理（ReKep、AffordVLA）、具身问答 EQA、世界模型（Cosmos）。

### 5.6 数据与基础设施
数据采集接口（UMI、ALOHA、FastUMI）、大数据集（Open X-Embodiment、DROID）、仿真 benchmark（ManiSkill、LIBERO、RLBench、Isaac Lab）、sim-to-real、真机自动评测。

### 5.7 与本项目的关系
- 已覆盖：抓取、拾放、装配（5.1）；选题 1/2/3 都在这一类。
- 方向 G = 5.3 移动操作，会自然牵入导航与任务规划。
- 小实验室常见进阶路线：**固定臂操作 → 移动操作 → 接 LLM 规划做长时程**，与本项目规划（阶段一二三+方向 G）吻合。

## 7. 概念讲解：多模态融合（Multimodal Fusion）

**定义**：把来自不同类型传感器/信息源的数据结合起来，做出比任何单一模态都更好的判断。模态 = 信息通道（图像/深度/语言/触觉/力觉/IMU…）。

### 7.1 动机：单模态各有盲区（互补）

| 模态 | 提供 | 盲区 |
|---|---|---|
| RGB | 颜色、纹理、语义 | 无距离；受光照影响 |
| 深度/点云 | 3D 几何、距离 | 无语义；**对黑色/透明/反光表面失效**（Gemini2 会遇到） |
| 语言 | 任务意图、常识 | 不精确、无坐标 |
| 触觉 | 接触位置、滑动、材质 | 仅接触瞬间有信息，感受野小 |
| 力/扭矩 | 接触力、碰撞、装配阻力 | 无空间信息 |
| IMU/编码器 | 自身运动状态 | 感知不到外部世界 |

例：SAM-6D = RGB 做语义分割（找零件）+ 点云做几何配准（算位姿），两模态各干擅长的事。

### 7.2 融合的三个层级（面试常考）

1. **早期融合（Data-level）**：原始数据拼接再进模型（如深度当第 4 通道拼成 RGBD）。学得到底层关联，但要求严格对齐、一个模态坏全坏。
2. **中期融合（Feature-level，主流）**：各模态独立编码器提特征，特征空间融合。手段：concat → 加权 → **交叉注意力（cross-attention，现在的标配）**——语言 token 作 query 注意图像 patch，模型自学"'红色零件'对应图像哪个区域"。CLIP、π0、OpenVLA 内部都如此。
3. **晚期融合（Decision-level）**：各模态独立出结果再投票/加权（如 RGB 位姿器与点云位姿器按置信度合并）。模块独立可降级，但丢底层关联。

另有**传统融合**：卡尔曼滤波/因子图按各传感器不确定性做最优估计（IMU+视觉里程计定位，SLAM 标准做法）。

### 7.3 融合前的两个对齐问题（工程上最花时间）

1. **空间对齐（标定）**：手眼标定（相机↔基座）、`depth_registration`（深度对齐 RGB 像素）都是。对不齐 = 张冠李戴。
2. **时间对齐（同步）**：传感器频率不同（相机 30Hz、力传感器 500Hz、关节 125Hz），按时间戳插值/就近匹配；ROS2 用 `message_filters`。

### 7.4 在具身智能中的形态

- **VLA = 视觉+语言→动作 的三模态融合**（π0 用 VLM backbone + 动作专家头）。
- **视觉+触觉**：Reactive Diffusion Policy（视觉慢层规划+触觉快层反应）、PolyTouch。
- **视觉+力觉**：精密装配经典组合——视觉引导到孔口，力控完成最后 1mm（与选题 1 直接相关）。
- **RGB+深度+语言**：开放词汇抓取，CLIP 特征投到点云做语义 3D 地图（VLMaps、OK-Robot）。

### 7.5 面试可答的本项目实例

1. RGB-D 融合：Gemini2 深度-彩色配准；SAM-6D 的"RGB 分割+点云配准"分工式融合。
2. 视觉+运动学融合：在手重估计时用"夹爪当前位姿（正运动学）"约束物体位姿。
3. 多假设融合：PPF 多候选 + ICP 打分筛选（决策级融合思想）。

补强建议：给 UR5 加腕部六维力传感器（或用 UR5 关节电流估计力）做**视觉+力觉装配**——既是选题 1 加分模块，又命中"多模态融合"JD 关键词。
