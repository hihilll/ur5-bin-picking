# 手眼标定结果验证清单（测试时照此执行）

> 配套 `docs/05_手眼标定.md`。标定完成、写入 `handeye_result.yaml` 并 `colcon build` 之后，
> 用本清单验证外参正确性。**标完必须验，验过才能进感知阶段。**
>
> 核心思想：相机看到的一个物理点，经外参 `base_link→camera` 换算到基座系后，
> 机器人走过去应准确到达该点。**贴合→外参对；偏移→外参错。**

---

## 前置

标定已完成并生效：
```bash
# .calib 已转成项目配置
python3 scripts/handeye_to_yaml.py
# 重编让静态 TF 生效
cd ros2_ws && colcon build --packages-select bin_picking_bringup && source install/setup.bash
```

起系统（发布 base_link→camera 静态 TF；不启抓取/感知，安全）：
```bash
ros2 launch bin_picking_bringup system.launch.py \
    enable_grasp:=false enable_perception:=false robot_ip:=192.168.0.11
```

> 按验证顺序执行：方法一 → 方法二 → 方法三。前两步不动机器人、几分钟完成，
> 能筛掉绝大多数问题；第三步才让机器人动、拿到最终毫米级精度。

---

## 方法一：点云目测对齐（最快，先做）

**目的**：5 秒钟筛掉"标歪了"的情况。

- [ ] RViz2：Fixed Frame 设 `base_link`
- [ ] Add → **PointCloud2**（相机点云）、**RobotModel**、**TF**
- [ ] 观察点云与机器人模型是否对齐：
  - 桌面点云应落在机器人模型的桌面高度
  - 料框点云应落在料框实际位置
  - **不应**悬空、穿模、或整体平移/旋转错位

**判定**
- ✅ 点云与模型贴合、位置合理 → 继续方法二
- ❌ 点云整体错位（如桌面点云飘在基座下方 10cm）→ 外参明显错，**直接重标**，不必往下验

---

## 方法二：坐标核对（定量，无运动，安全）

**目的**：不动机器人，定量核对"相机点→基座点"换算是否准确。

```bash
ros2 run bin_picking_bringup handeye_verify
```

- [ ] RViz2：Add → **Marker**（话题 `/handeye_verify/marker`）
- [ ] 工具栏点 **Publish Point**，点场景中一个**已知/可量的物理点**
      （标定板角点、桌上尖点、料框角）
- [ ] 看 `handeye_verify` 终端打印：
      ```
      点 [camera_link] (...) -> [base_link] (x, y, z)
      ```
- [ ] RViz 在换算出的基座系坐标处显示**红球**

**判定**
- ✅ 红球**贴在你点的那个物理点的点云上** → 外参 OK，继续方法三
- ✅ 若能量出该点相对基座的真实坐标，与终端打印的 `(x,y,z)` 对比，差值应很小
- ❌ 红球明显飘开点云 → 外参有偏差，见下方"偏差定位"

---

## 方法三：机器人实走量偏差（终验，毫米级）

**目的**：拿到量化精度。⚠️ 让机器人真运动，务必按安全要求做。

`handeye_verify` 点完物理点后会发布 `/handeye_verify/target_pose`
（夹爪竖直向下、对准该点）。

- [ ] **把速度缩放调到 0.1（很慢）**
- [ ] RViz **MotionPlanning** 面板，目标设成 `/handeye_verify/target_pose`
- [ ] ⚠️ 先把目标**抬高几 cm**悬停在物理点上方，避免 TCP 真撞到点/桌面
- [ ] **手握急停**，Plan → 检查轨迹合理 → Execute
- [ ] 机器人 TCP 到位后，**用尺子/卡尺量 TCP 尖端与真实物理点的偏差**

**判定**
- ✅ 偏差 **≤ 几毫米** → 标定合格，可进入感知阶段
- ❌ 偏差 **≥ 1~2cm** → 标定不准，重标

---

## 偏差大时定位原因

| 偏差表现 | 最可能原因 | 处理 |
|---|---|---|
| 各方向**恒定偏一个固定量** | `marker_size` 量错（最常见）；或相机内参不准 | 卡尺重量图案边长、改 `marker_size` 重标 |
| 每次点不同点，偏差**方向/大小都不一样** | 采样姿态太少/太集中，标定不稳 | 重采，加大姿态多样性（大角度、覆盖广，15~20 组） |
| 红球**整体旋转错位** | 外参旋转分量错，通常也是采样多样性不足 | 同上，重采 |
| verify 报 **TF 转换失败** | 手眼静态 TF 没发；或点的 frame 不对 | 确认 `system.launch` 在跑、`handeye_result.yaml` 已生效 |

---

## 安全提醒（方法三）

- 采样/实走全程**低速 + 手握急停**
- Plan 完先看 RViz 轨迹合理再 Execute
- 第一次 Execute 用最低速度缩放（0.1）
- 目标先悬停在物理点上方，别直接怼到点/桌面

---

## 通过标准（一句话）

**方法一点云对齐 + 方法二红球贴合点云 + 方法三实走偏差 ≤ 毫米级** → 手眼标定通过，
进入 `docs/03_测试步骤.md` 的感知测试。
