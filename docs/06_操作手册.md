# 操作手册（从编译完成到整机跑通 · 命令速查）

> 按顺序做，任何一步不过就停下排查，别急着往下。
> 详细说明分别见：标定 `docs/05`，测试 `docs/03`，运行 `docs/02`，夹爪URDF `docs/04`。
> ⚠️ 涉及真机运动：手握急停、周围无人、**第一次速度调到最低**、先空跑不放零件。

## 前置（每个新终端都要执行）
```bash
cd ~/ros2_ws && source install/setup.bash
```

---

## 第 0 步：无硬件仿真自检
```bash
# 终端A
ros2 launch bin_picking_bringup sim.launch.py
# 终端B
ros2 run bin_picking_grasp publish_test_grasp
ros2 service call /pick_place/run std_srvs/srv/Trigger {}
```
预期：终端A 依次打印 开夹爪→预抓取→抓取→闭合→抬起→放置→退回，全部成功。

```bash
ros2 launch orbbec_camera gemini2.launch.py

ros2 launch ur_robot_driver ur_control.launch.py \
    ur_type:=ur5 robot_ip:=<IP> launch_rviz:=true
# 新终端
ros2 topic echo /joint_states --once
```

## 第 3 步：夹爪
```bash
# 先模拟
ros2 run bin_picking_grasp gripper_driver --ros-args -p simulate:=true
ros2 service call /gripper/set_gripper bin_picking_interfaces/srv/SetGripper \
    "{width: 0.05, force: 50.0, speed: 50.0}"
# 真机（需先填好 gripper_driver.py 的 REG_*）
ls /dev/ttyUSB*
ros2 run bin_picking_grasp gripper_driver --ros-args -p simulate:=false -p port:=/dev/ttyUSB0
ros2 service call /gripper/set_gripper bin_picking_interfaces/srv/SetGripper "{width: 0.0, force: 30.0, speed: 30.0}"
```

## 第 4 步：手眼标定 ★

### 4.1 装依赖 + 重编
```bash
sudo apt install ros-humble-easy-handeye2 ros-humble-aruco-ros
cd ~/ros2_ws && colcon build --packages-select bin_picking_bringup && source install/setup.bash
```
### 4.2 准备：ArUco 单码贴末端法兰，量准边长（如 0.06m），记 id（默认26）

### 4.3 起前置（两个终端）
```bash
ros2 launch orbbec_camera gemini2.launch.py
ros2 launch ur_robot_driver ur_control.launch.py ur_type:=ur5 robot_ip:=<IP>
# 检查
ros2 topic hz /camera/color/image_raw
ros2 run tf2_ros tf2_echo base_link tool0
```
### 4.4 启动标定
```bash
ros2 launch bin_picking_bringup calibrate_handeye.launch.py marker_id:=26 marker_size:=0.06
# 若彩色话题名不同：加 image_topic:=... camera_info_topic:=...
# 确认检测到码：
ros2 topic echo /aruco_single/pose
ros2 run tf2_ros tf2_echo camera_link handeye_marker
```
### 4.5 采样（easy_handeye2 GUI）
手动低速换姿态 → Take Sample，采 15~20 组（大角度、覆盖视野）→ Compute → Save
（存到 `~/.ros2/easy_handeye2/handeye_ur5_gemini2.calib`）

### 4.6 写入配置 + 重编
```bash
cd ~/robotarm        # 代码根目录
python3 scripts/handeye_to_yaml.py
cd ros2_ws && colcon build --packages-select bin_picking_bringup && source install/setup.bash
```
### 4.7 精度验证（无自动运动，安全）
```bash
ros2 launch bin_picking_bringup system.launch.py robot_ip:=<IP> enable_grasp:=false enable_perception:=false
# 新终端
ros2 run bin_picking_bringup handeye_verify
```
RViz：Fixed Frame=base_link，Add PointCloud2 + Marker(/handeye_verify/marker)；
用 Publish Point 点已知物理点 → 看红球 → MotionPlanning 用 /handeye_verify/target_pose 低速走过去 → 量误差应毫米级。

## 第 5 步：CAD + 感知
```bash
# CAD 放到 ros2_ws/src/bin_picking_description/meshes/part.stl，重编 description
# （可选，Windows 先验证算法）
python scripts/offline_pose_test.py --model part.stl --scale 0.001

# 真机感知
ros2 launch orbbec_camera gemini2.launch.py
ros2 launch bin_picking_perception perception.launch.py \
    cad_model_path:=$(ros2 pkg prefix bin_picking_description)/share/bin_picking_description/meshes/part.stl
ros2 topic echo /detected_objects
```
离线调参（先录后回放，不用一直开相机）：
```bash
bash scripts/record_pointcloud.sh bin_scene_01
bash scripts/replay_and_perceive.sh bin_scene_01 /abs/part.stl
```
调参改 `perception_params.yaml`：workspace_* → voxel_size → cluster_* → min_fitness

## 第 6 步：抓取标注 + 规划
```bash
# 标注（Windows 可跑）：Shift+左键点选 → 复制输出到 grasp_params.yaml 的 grasp_annotations
python scripts/annotate_grasp.py --model part.stl --scale 0.001 --width 0.03

# 规划验证（需感知 + 手眼TF 已起）
ros2 run bin_picking_grasp grasp_planner --ros-args \
    --params-file $(ros2 pkg prefix bin_picking_grasp)/share/bin_picking_grasp/config/grasp_params.yaml
ros2 topic echo /grasp_candidates
# RViz 看 /grasp_markers（绿箭头=最佳抓取）
```
