# 08 阶段二：大模型感知接入（SAM-6D / FoundationPose / 学习型抓取）

> 依据 2024–2025 文献调研（BOP 2024 报告、OpenCV BPC 2025 工业挑战赛、Pickalo 等）选型：
> - **料框内位姿**：SAM-6D（分割+位姿一体、零样本、位姿网络 ~57ms、开源免 license）
> - **在手位姿（检视位）**：FoundationPose track 模式（~30ms，对夹爪遮挡鲁棒）
> - **学习型抓取（可选路线）**：HGGD / EconomicGrasp（开源，GraspNet-1B 权重）；AnyGrasp 需 license，备选
>
> GPU 按 RTX 4090 (24GB) 规划，三模型可**常驻并存**（SAM-6D ~6G + FoundationPose ~8G + 抓取网络 ~3G）。

---

## 1. 架构

```
┌── ROS2 侧（系统 Python，colcon 构建）────────┐   ┌── 推理侧（conda/Docker，独立环境）──┐
│ bin_picking_perception_v2:                    │   │ inference/server.py                  │
│   sam6d_client ──────/detected_objects──→ 下游│ZMQ│   wrappers/sam6d_wrapper.py          │
│   foundationpose_client ─/detected_objects_   │←─→│   wrappers/foundationpose_wrapper.py │
│                            inhand──→ inhand_  │   │   wrappers/grasp_wrapper.py          │
│   grasp_client ──────/grasp_candidates──→ 下游│   │   wrappers/fake_wrapper.py (--fake)  │
└───────────────────────────────────────────────┘   └──────────────────────────────────────┘
```

- 客户端只做「订阅 RGB-D → ZMQ 请求 → 发 ROS 话题」，**所有模型依赖都在推理侧**，
  ROS 环境零污染；推理服务崩溃只影响感知输出，机器人侧安然无恙。
- 话题接口与阶段一完全一致，`grasp_planner / grasp_executor / pick_loop / inhand_estimator` 零改动。
- 依赖冲突时可拆分：`server.py --models sam6d --port 5555`、
  `--models foundationpose --port 5556`（FoundationPose 官方 Docker 内）、
  `--models grasp --port 5557`，再改 `perception_v2_params.yaml` 各自的 `server_address`。

### 两条抓取路线（system_v2.launch.py 二选一）
| 路线 | 链路 | 适用 |
|---|---|---|
| A 位姿路线（默认） | sam6d_client → grasp_planner（CAD 标注抓取点） | 可控制抓取面向，利于装配 |
| B 学习抓取路线 | grasp_client 直接出 `/grasp_candidates` | 杂乱堆叠更鲁棒，不保证面向 |

在手补偿（`enable_inhand:=true`）两条路线都能叠加；路线 B + 在手补偿 = 方案设计里的推荐组合。

## 2. 第一步：不装任何模型，先打通链路（无 GPU 也可做）

```bash
# 终端1: 假模式推理服务（只需 pip install pyzmq numpy）
cd ~/robotarm/inference && python3 server.py --fake

# 终端2: 起相机（或回放 rosbag: bash scripts/replay_and_perceive.sh <bag>）
ros2 launch orbbec_camera gemini2.launch.py

# 终端3: 起 v2 感知客户端
ros2 launch bin_picking_perception_v2 perception_v2.launch.py \
    enable_sam6d:=true enable_grasp:=true

# 验证: 应看到固定假位姿/假抓取
ros2 topic echo /detected_objects --once
ros2 topic echo /grasp_candidates --once
```

通过后再逐个接真模型（每接一个，同样方法单独验证）。

⚠️ **深度对齐**：客户端假设 `depth_topic` 是**对齐到彩色**的深度图。
orbbec 驱动需开启对齐（launch 参数 `depth_registration:=true` 或
`align_mode`，以所装 OrbbecSDK_ROS2 版本 README 为准），并把
`perception_v2_params.yaml` 的 `depth_topic` 改成实际话题名。

## 3. SAM-6D 安装与验证（料框位姿，主力）

```bash
git clone https://github.com/JiehongLin/SAM-6D
cd SAM-6D
# 按官方 README: conda create -n sam6d python=3.9 && 装 pytorch(CUDA12.x for 4090)
# 编译 pointnet2；下载 ISM(SAM ViT-H 或 FastSAM) 与 PEM 权重
```

1. **CAD 准备**：零件 CAD 转 **mm 单位** `.ply`（SAM-6D 官方约定，内部会 /1000）。
2. **渲染模板**（一次性，官方 blenderproc 脚本）：
   ```bash
   blenderproc run render_custom_templates.py \
       --output_dir /abs/render_out --cad_path /abs/part_mm.ply
   # 模板生成在 /abs/render_out/templates/
   ```
3. **启动与验证**（wrapper 已写好，`[V1]~[V5]` 核对点见 `sam6d_wrapper.py`）：
   ```bash
   python3 server.py --models sam6d \
       --sam6d-repo ~/SAM-6D/SAM-6D --sam6d-cad /abs/part_mm.ply \
       --templates /abs/render_out/templates
   ros2 launch bin_picking_perception_v2 perception_v2.launch.py
   # RViz: /detected_markers 的球应落在零件上; TF object_0 姿态应贴合
   ```
   离线调参可配合录包回放（`scripts/replay_and_perceive.sh`）。
   已知实现细节：ISM 与 PEM 两个子目录有同名 `utils` 包（官方分进程跑），
   wrapper 里用 sys.modules 清缓存的方式在同进程加载两者；若现场仍串包，
   看 `sam6d_wrapper.py` 顶部注释里的退路方案。

## 4. FoundationPose 安装与验证（在手位姿）

```bash
git clone https://github.com/NVlabs/FoundationPose
# 推荐官方 Docker（nvdiffrast/kaolin 编译麻烦）:
cd FoundationPose && bash docker/run_container.sh
# 容器内: 下载权重，先跑通官方 run_demo.py
```

1. 在容器内 `pip install pyzmq`，把 `inference/` 目录挂载进容器。
2. **CAD 准备**：FoundationPose 期望 **米单位** 网格（mm 模型先缩放 0.001 另存；
   wrapper 会检查尺寸 >1m 时报错提醒）。
3. 启动（wrapper 已写好，含深度带 mask + 跳变检测 + 自动重 register）：
   ```bash
   python3 server.py --models foundationpose --port 5556 \
       --fp-repo /path/FoundationPose --fp-cad /abs/part_m.obj \
       --inspect-zmin 0.25 --inspect-zmax 0.70   # 检视位深度带，按实际距离调
   # 记得改 perception_v2_params.yaml 里 foundationpose_client 的 server_address
   ```
4. 验证：手持零件在相机前移动，`/detected_objects_inhand` 位姿应平滑跟随；
   每抓一个新零件后调 `ros2 service call /inhand_pose/reset std_srvs/srv/Trigger {}`。
   若 register 位姿飘：多半是深度带混入夹爪指尖 → 收紧 `--inspect-zmin/zmax`。

## 5. 学习型抓取网络（路线 B）

wrapper 按 **graspnet-baseline 系 API** 写好（EconomicGrasp 同源，改网络构造即可）：
```bash
git clone https://github.com/graspnet/graspnet-baseline   # 先跑通这个最简
# 装环境(含 pointnet2/knn 编译 + graspnetAPI) + 下载官方 GraspNet-1B 权重(checkpoint-rs.tar)
python3 server.py --models grasp --port 5557 \
    --grasp-repo ~/graspnet-baseline --grasp-ckpt /abs/checkpoint-rs.tar
```
升级选项：EconomicGrasp（改 `load()` 里的网络构造 `[V1]`/`[V4]`）；
AnyGrasp 若申请到 license 同样只动 `load()/infer()` 两处。

⚠️ **坐标系契约（最容易错）**：graspnetAPI 约定抓取 **x 轴 = 接近方向**，
本项目执行器约定 **z 轴 = 接近方向**（沿 z 回退出预抓取位）。
`grasp_wrapper.py` 的 `AXIS_CONV` 已做转换、平移已沿接近轴推进 `g.depth`
到指尖抓取点（`[V3]`）——集成后**必须在仿真里核对一次**：
预抓取位应在抓取位的接近轴反方向上（俯视抓取即正上方），确认无误再上真机。

## 6. 全系统真机运行

```bash
# 终端1: 推理服务（全量或分实例）
python3 inference/server.py --cad ... --templates ... --grasp-ckpt ...

# 终端2: 路线A（位姿+标注抓取）
ros2 launch bin_picking_bringup system_v2.launch.py robot_ip:=192.168.0.11
# 或 路线B（学习抓取）+ 在手补偿:
ros2 launch bin_picking_bringup system_v2.launch.py robot_ip:=192.168.0.11 \
    enable_pose_route:=false enable_learned_grasp:=true enable_inhand:=true

# 终端3: 触发
ros2 service call /pick_place/run std_srvs/srv/Trigger {}     # 单次
ros2 service call /bin_picking/start std_srvs/srv/Trigger {}  # 循环
```

## 7. 故障排查

| 现象 | 排查 |
|---|---|
| 客户端持续报「推理服务无响应」 | server.py 是否在跑？端口/地址是否与 yaml 一致？防火墙？ |
| 位姿明显错位 | 深度未对齐彩色（第2节）；CAD 单位 mm/m 混淆；手眼外参未标定 |
| 抓取箭头方向朝外/横着 | graspnetAPI 轴约定未转换（第5节契约） |
| register 后 track 飘 | 检视位深度 mask 混入夹爪指尖 → 收紧深度带或换 ISM mask |
| 显存不足 | `nvidia-smi` 看占用；按第1节拆分实例分时启动 |
| pick_loop 报无候选 | `/grasp_candidates` 时效 2s（candidate_timeout），推理太慢时调大 |

## 8. 已知边界

- SAM-6D 全流程（ISM 分割 ~1s + PEM ~57ms）单帧秒级，`process_every_n` 已调低频；
  bin picking 每抓一次识别一次，够用。
- 强反光/超小零件（<2cm）深度质量差时上限在深度而非模型 —— 后续可考虑
  FoundationStereo 类深度修复（暂不引入）。
- 本骨架用 pickle+ZMQ，仅限本机/可信内网。
