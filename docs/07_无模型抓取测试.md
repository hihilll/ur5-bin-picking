# 07 无模型抓取测试（无 CAD，抓螺丝螺母等小物件）

手头没有 3D 打印件 / 没有 CAD 模型时，用这条**平行链路**跑通整套硬件闭环
（相机→手眼标定→抓取→MoveIt→夹爪），验证系统联通性。

> 与阶段2(基于模型)互不影响：阶段2 用 `perception_node + grasp_planner`(需 CAD)；
> 本方案用单个 `model_free_grasp` 节点，直接从点云算**俯视二指抓取**，
> 发到同一个 `/grasp_candidates`，下游 `grasp_executor / pick_loop / gripper_driver` 不改。

## 原理
点云 →(TF 转基座系)→ 裁剪工作区 → 去桌面/框底平面 → DBSCAN 聚类。
每个物体簇：
- 抓取位置 = 质心(x,y)，z = 簇顶 − `grasp_depth`
- 接近方向 = 竖直向下
- 手指闭合方向 = 物体水平投影的**短边**(PCA 次主轴)，即夹住较窄一侧
- 开口宽度 = 短边尺寸 + `width_margin`，超行程则丢弃
- 打分：越靠上(堆顶)越优先

## 前置
1. **手眼标定必须先做**（`docs/05`），否则相机点云无法正确转到基座系，抓取位置全错。
2. 参数按你的桌面/物体调：`config/model_free_params.yaml`（在 `bin_picking_perception`）。
   - `workspace_min/max`：**基座系**下你摆物体的范围（先粗设，再按 RViz 里点云位置收紧）。
   - 小物件建议 `voxel_size:0.002`、`cluster_eps:0.008`、`cluster_min_points:30`（默认已设）。

## 用法

### A. 先只看抓取候选，不动机械臂（强烈建议第一步）
```bash
ros2 launch bin_picking_bringup model_free_test.launch.py \
    enable_robot:=false enable_moveit:=false
```
RViz 里加 `/grasp_markers`(MarkerArray)：绿色箭头=最佳抓取。确认箭头落在物体上、
数量合理，再进下一步。也可单独起节点调参：
```bash
ros2 run bin_picking_perception model_free_grasp \
    --ros-args --params-file <path>/model_free_params.yaml
```

### B. 全链路（低速、手握急停）
```bash
ros2 launch bin_picking_bringup model_free_test.launch.py robot_ip:=192.168.0.11
# 无真实夹爪时：grasp_params.yaml 里 gripper_driver.simulate: true
```
- 单次取放：`ros2 service call /pick_place/run std_srvs/srv/Trigger {}`
- 自动循环：`ros2 service call /bin_picking/start std_srvs/srv/Trigger {}`

## 注意
- **TCP 标定**：`grasp_params.yaml` 的 `tcp_link` 默认 `tool0`(法兰)。装了夹爪后应改成
  夹爪指尖坐标系(见 `docs/04`)，否则抓取深度会差一个夹爪长度。
- `place_pose`(基座系 TCP 放置位姿) 随便设个安全位置即可，无模型放置不需要 CAD。
- 局限：只做俯视抓取；对超出夹爪行程(默认>50mm 短边)或又大又平的物体会跳过。
- 想抓更小的螺丝：减小 `voxel_size`、`cluster_eps`、`cluster_min_points`。
