# 09 整机测试教程（硬件 + 全流程命令速查）

> 本文是**一站式教程**：从硬件清单、安装接线，到每一步测试的命令与判定标准，
> 覆盖 阶段一(传统) / 无模型 / 阶段二(大模型) 三条链路。
> 单项细节见对应文档：环境 `01`、标定 `05`、夹爪 `04`、无模型 `07`、大模型 `08`。

# 第一部分：硬件

## 1.1 硬件清单

| 硬件 | 型号/规格 | 接口 | 用途 |
|---|---|---|---|
| 机械臂 | UR5（负载 5kg，臂展 850mm） | 以太网（控制柜 LAN 口） | 执行取放 |
| 深度相机 | 奥比中光 Gemini2（深度 0.15~10m，USB 3.0） | USB 3.0 Type-C（务必接 USB3 口） | 点云/RGB-D 感知 |
| 夹爪 | 沃姆 EPGC-50-150-B-C-NM-L2（行程 50mm，负载 3kg） | USB（内部 Modbus-RTU 串口）+ 独立供电（按手册，常见 DC 24V） | 抓取 |
| 上位机 | Ubuntu 22.04 + ROS2 Humble；阶段二需 GPU（按 RTX 4090 24G 规划） | — | 运行全部软件 |
| 料框 | 浅色**哑光**塑料框（避免黑色吸光/反光材质），尺寸参考 400×300×100mm | — | 放零件 |
| 标定板 | ArUco 单码（默认 id=26），打印边长 **0.06m**，贴硬质平板 | — | 手眼标定 |
| 网络 | 交换机/路由器 或 网线直连 | — | PC ↔ UR5 控制柜 |
| 零件 | 3D 打印件（设计时避免完全对称，带可区分特征） | — | 被抓对象 |

## 1.2 物理布局（Eye-to-Hand）

```
            ┌────────┐  相机支架（固定，不随臂动）
            │Gemini2 │  ← 镜头竖直向下，正对料框中心
            └───┬────┘     推荐高度：料框上方 0.6~0.8m
                │           （视野盖住整框 + 深度精度较好的区间）
   ┌────────────▼───────────┐
   │        料框(零件)        │   料框放在 UR5 可达范围内
   └────────────────────────┘   （基座前方 0.3~0.7m 为舒适区）
        ┌──────┐
        │ UR5  │ 基座
        └──────┘   法兰(tool0) → 转接板 → 沃姆夹爪（竖直朝下姿态可达料框全域）
```

布局要点：
- **相机必须刚性固定**（型材支架/横梁），标定后任何位移都要重标。支架避开机械臂运动空间。
- 相机视野同时盖住：料框全域 + **检视位**（阶段四在手重估计时零件被举到的位置，
  默认 `inspection_pose` 在基座前方 0.3m、高 0.5m，可在 `grasp_params.yaml` 改）。
- 放置区与料框分开，均在臂可达范围；操作员位置能随手够到**急停**。
- 灯光：稳定漫射光，避免强烈直射光/频闪（影响深度与 ArUco 检测）。

## 1.3 接线与连接拓扑

```
 PC(Ubuntu) ──网线── 交换机 ──网线── UR5 控制柜 LAN
     │ USB3 ── Gemini2 相机
     │ USB  ── 沃姆夹爪（虚拟串口 /dev/ttyUSB0）    夹爪另接 DC 供电（按手册）
     │ (阶段二) 本机 4090 GPU
```

| 连接 | 检查命令 | 预期 |
|---|---|---|
| UR5 网络 | `ping 192.168.0.11` | 通 |
| 相机 USB3 | `lsusb`（装完 udev 规则后查 Orbbec 设备） | 识别到设备 |
| 夹爪串口 | `ls /dev/ttyUSB*` | 出现 /dev/ttyUSB0（拔插对比确认是哪个） |
| GPU | `nvidia-smi` | 驱动正常、显存 24G |

## 1.4 网络配置

1. **UR5 侧**（示教器）：设置 → 网络 → 静态 IP `192.168.0.11`，子网掩码 `255.255.255.0`。
2. **PC 侧**：有线网卡设静态 IP，同网段不同址（如 `192.168.0.100`）。
3. **External Control URCap**（关键）：示教器安装 External Control URCap，
   程序里 Host IP 填 **PC 的 IP**（192.168.0.100），端口默认 50002。
   每次要控制机械臂，都要在示教器**运行这个 External Control 程序**。
4. 已知问题：本机网络延迟偏大，UR 驱动启动必须加
   `rtde_config_package_timeout:=500`，否则 RTDE 握手超时、驱动 SIGABRT
   （项目 launch 已内置默认值，手动起 `ur_control.launch.py` 时别忘）。

## 1.5 夹爪机械与电气安装

1. **机械**：转接板连接 UR5 法兰(tool0) 与夹爪本体；记录（后续填 URDF，见 `04`）：
   - 转接板厚度 → `ur5_with_gripper.xacro` 挂载 `origin.z`
   - 夹爪本体长、指长 → `worm_epgc50.xacro` 的 `body_len` / `finger_len`
   - 安装朝向 → 挂载 `origin.rpy`
2. **电气**：夹爪独立供电（电压按手册）；USB 线接 PC。
3. **软件前提**：拿到沃姆 Modbus 协议手册后，填 `gripper_driver.py` 顶部的
   `REG_*` 寄存器地址与 `POS_COUNT_MAX`（当前是占位值，**没填之前只能用 simulate 模式**）。
4. 串口权限：`sudo usermod -aG dialout $USER` 后**重新登录**。

## 1.6 标定板制作与安装

- 生成 ArUco 单码（原始 ARUCO 字典，id=26），打印后**实measure 边长**（必须与
  launch 参数 `marker_size` 一致，差 1mm 标定就差 1mm 级）。
- 贴在硬质平板上（不可弯曲），标定时**刚性固定在末端法兰**（夹爪夹持或螺栓固定均可，
  标定过程中不得松动）。

## 1.7 安全须知（每次真机测试前过一遍）

- 手握急停；周围无人、无障碍物；示教器速度滑块调到**最低**（如 10%）。
- 第一次跑任何新轨迹：先不放零件空跑；确认轨迹后再逐步提速。
- 夹爪先 `simulate:=true` 走通逻辑，再上真夹爪。
- 阶段二学习型抓取第一次上真机前，必须先在仿真里核对抓取方向（见 3.4 节）。

# 第二部分：软件测试全流程（阶段一 + 无模型）

> 环境搭建（装 ROS2/驱动/MoveIt/easy_handeye2）见 `01_环境搭建.md`，此处从编译开始。
> 每个新终端先执行：`cd ~/ros2_ws && source install/setup.bash`

## 2.0 编译自检（不接任何硬件）

```bash
cd ~/ros2_ws
rosdep install --from-paths src --ignore-src -r -y
pip install open3d opencv-contrib-python numpy scipy pymodbus pyserial pyzmq
colcon build --symlink-install
source install/setup.bash

# 判定
ros2 pkg list | grep bin_picking          # 应列出 6 个包（含 perception_v2）
ros2 interface show bin_picking_interfaces/msg/GraspCandidate
```
一键版：`bash scripts/check_build.sh`

## 2.1 无硬件仿真（验证取放逻辑）

```bash
# 终端A：fake hardware UR5 + MoveIt + 模拟夹爪 + 执行器
ros2 launch bin_picking_bringup sim.launch.py
# 终端B：喂一个固定抓取，触发一次取放
ros2 run bin_picking_grasp publish_test_grasp
ros2 service call /pick_place/run std_srvs/srv/Trigger {}
```
**判定**：终端A 依次打印 开夹爪→预抓取→抓取→闭合→抬起→放置→退回，RViz 里机械臂动。

## 2.2 相机单测

```bash
ros2 launch orbbec_camera gemini2.launch.py
# 新终端
ros2 topic list | grep -E "points|color|depth"     # ★记下真实话题名
ros2 topic hz /camera/depth_registered/points
rviz2    # Fixed Frame=camera_link, Add PointCloud2 看料框点云
```
**判定**：点云稳定 ~15-30Hz，RViz 能看到料框与零件。
★ 话题名与配置不一致时，改：`perception_params.yaml` 的 `input_topic`、
`model_free_params.yaml` 的 `input_topic`、`perception_v2_params.yaml` 的三个 topic。

## 2.3 UR5 单测

```bash
ros2 launch ur_robot_driver ur_control.launch.py \
    ur_type:=ur5 robot_ip:=192.168.0.11 launch_rviz:=true \
    rtde_config_package_timeout:=500
# 示教器运行 External Control 程序，然后：
ros2 topic echo /joint_states --once
```
**判定**：读到 6 关节角；手动拖真实臂，RViz 模型跟随。

## 2.4 夹爪单测

```bash
# 模拟（不接硬件）
ros2 run bin_picking_grasp gripper_driver --ros-args -p simulate:=true
ros2 service call /gripper/set_gripper bin_picking_interfaces/srv/SetGripper \
    "{width: 0.05, force: 50.0, speed: 50.0}"       # 期望 success:true

# 真机（前提：REG_* 已按沃姆手册填好）
ls /dev/ttyUSB*
ros2 run bin_picking_grasp gripper_driver --ros-args -p simulate:=false -p port:=/dev/ttyUSB0
ros2 service call /gripper/set_gripper bin_picking_interfaces/srv/SetGripper "{width: 0.0, force: 30.0, speed: 30.0}"   # 全闭
ros2 service call /gripper/set_gripper bin_picking_interfaces/srv/SetGripper "{width: 0.05, force: 30.0, speed: 30.0}"  # 全开
```
**判定**：真夹爪开合到位、方向正确（反了调 `width_to_counts`/`POS_COUNT_MAX`）。

## 2.5 手眼标定 ★（一切位姿的基础）

完整流程见 `05_手眼标定.md`，速查：
```bash
sudo apt install ros-humble-easy-handeye2 ros-humble-aruco-ros
# 前置：相机 + UR 驱动已起（2.2/2.3 的两个终端保持运行）
ros2 launch bin_picking_bringup calibrate_handeye.launch.py marker_id:=26 marker_size:=0.06
ros2 topic echo /aruco_single/pose        # 确认检测到码
# GUI 里：变换臂姿态(大角度、覆盖视野) → Take Sample ×15~20 → Compute → Save
python3 ~/robotarm/scripts/handeye_to_yaml.py
cd ~/ros2_ws && colcon build --packages-select bin_picking_bringup && source install/setup.bash
```
**精度验证（必做）**：
```bash
ros2 launch bin_picking_bringup system.launch.py robot_ip:=192.168.0.11 \
    enable_grasp:=false enable_perception:=false
ros2 run bin_picking_bringup handeye_verify
# RViz(Fixed Frame=base_link)：Publish Point 点一个已知物理点 → 出红球
# → MotionPlanning 以 /handeye_verify/target_pose 为目标低速走过去 → 量偏差
```
**判定**：TCP 实际到达点与物理点偏差**毫米级**。偏差 >5mm 重标。

## 2.6 感知单测（阶段一 PPF+ICP，需 CAD）

```bash
# （可选，Windows 先离线验证算法）python scripts/offline_pose_test.py --model part.stl --scale 0.001
# CAD 放 ros2_ws/src/bin_picking_description/meshes/part.stl，重编 description 包
ros2 launch orbbec_camera gemini2.launch.py
ros2 launch bin_picking_perception perception.launch.py \
    cad_model_path:=$(ros2 pkg prefix bin_picking_description)/share/bin_picking_description/meshes/part.stl
ros2 topic echo /detected_objects
```
**判定**：RViz 中绿色球贴在零件上、TF `object_0` 朝向合理，fitness ≥0.4。
调参顺序（`perception_params.yaml`）：`workspace_*` → `voxel_size` → `cluster_*` → `min_fitness`。
离线调参：`bash scripts/record_pointcloud.sh scene01` 录包 →
`bash scripts/replay_and_perceive.sh scene01 /abs/part.stl` 反复回放。

## 2.7 抓取标注 + 规划单测（不动机器人）

```bash
# 标注（Windows 可跑）：Shift+左键点选抓取点，输出粘贴到 grasp_params.yaml 的 grasp_annotations
python scripts/annotate_grasp.py --model part.stl --scale 0.001 --width 0.03

# 规划（需 2.6 感知 + 手眼TF 在跑）
ros2 run bin_picking_grasp grasp_planner --ros-args \
    --params-file $(ros2 pkg prefix bin_picking_grasp)/share/bin_picking_grasp/config/grasp_params.yaml
ros2 topic echo /grasp_candidates
```
**判定**：RViz `/grasp_markers` 绿箭头（最佳抓取）指向零件、适合从上方接近。

## 2.8 整机干跑（真臂低速，先不放零件）

```bash
ros2 launch bin_picking_bringup system.launch.py \
    robot_ip:=192.168.0.11 cad_model_path:=/abs/.../meshes/part.stl
# 夹爪没就绪：grasp_params.yaml 设 gripper_driver.simulate: true
ros2 service call /pick_place/run std_srvs/srv/Trigger {}
```
**判定**：UR5 低速走完 预抓取→抓取→抬起→放置 全轨迹，不碰料框。
待填参数：`grasp_params.yaml` 的 `place_pose`、`bin_center/bin_size`（料框实测）、
装好夹爪后 `tcp_link` → `gripper_grasp_tcp`（并按 `04` 集成 URDF/SRDF）。

## 2.9 整机闭环（放真零件）

```bash
ros2 launch bin_picking_bringup system.launch.py robot_ip:=192.168.0.11 cad_model_path:=/abs/part.stl
ros2 service call /bin_picking/start std_srvs/srv/Trigger {}     # 自动循环
ros2 topic echo /bin_picking/status                              # 看状态
ros2 service call /bin_picking/stop std_srvs/srv/Trigger {}      # 随时停
```
**判定（阶段一验收）**：从无序堆叠稳定抓取并放置，连抓 20 次统计成功率；
框空后状态自动变 `DONE`。高精度装配再开 `enable_inhand: true` + 填 `inspection_pose`。

## 2.10 无模型链路（无 CAD，抓螺丝螺母等，可先于 2.6~2.9 做）

```bash
# 只看抓取候选（不动臂）：

    robot_ip:=192.168.0.11 enable_robot:=false enable_moveit:=false
```

## 3.1 假模式通链路（无 GPU/模型即可做）

```bash

    --grasp-repo ~/graspnet-baseline --grasp-ckpt /abs/checkpoint-rs.tar

    enable_sam6d:=false enable_grasp:=true
```

**判定 1**：RViz `/grasp_markers` 箭头落在物体上。
**判定 2（必做，上真机前）**：在**仿真**里核对抓取方向——
```bash
ros2 launch bin_picking_bringup sim.launch.py     # 另一台或先关真机驱动
# 用 grasp_client 的输出触发 /pick_place/run，观察 RViz：
# 预抓取位必须在抓取位的接近轴反向（俯视抓取=正上方 10cm）。
# 若方向错乱 → grasp_wrapper.py 的 AXIS_CONV/depth 推进需按实测调（[V3]）
```


| 现象 | 最可能原因 → 处理 |

| UR 驱动启动即 Aborted(-6) | RTDE 握手超时 → 加 `rtde_config_package_timeout:=500` |
