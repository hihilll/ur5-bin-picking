# 阶段1：手眼标定（Eye-to-Hand / easy_handeye2）

> 目标：解出 **base_link → camera_link** 外参并填入
> `bin_picking_bringup/config/handeye_result.yaml`，让"相机看到的点，机器人能准确走到"。
> 这是后续所有真机测试（感知→抓取→放置）的前提。

## 0. 原理与布局

- **Eye-to-Hand**：相机固定俯视料框（不装在手上）。对应 easy_handeye2 的
  `calibration_type = eye_on_base`。
- 把 **ArUco 标定板贴在机械臂末端法兰**，机器人带着它在相机视野内摆多个姿态；
  easy_handeye2 采集 `base_link→tool0`（正解）与 `camera_link→marker`（视觉检测）
  多组配对，解出相机相对基座的固定外参。
- 本项目把标定的 `tracking_base_frame` 设为 `camera_link`（而非光学系），
  所以求得的结果**直接就是** `base_link→camera_link`，与 `handeye_result.yaml` 对齐，
  也不会和相机驱动内部 TF 打架。

## 1. 准备

```bash
sudo apt install ros-humble-easy-handeye2 ros-humble-aruco-ros
```
- 打印一张 **ArUco 单码**（默认字典 DICT，id 可选，本项目默认 `marker_id:=26`）。
- **量准边长**（例 60mm=0.06m），必须与 `marker_size` 参数一致，否则外参有系统误差。
- 把码平整固定在末端法兰上（别翘边、别反光）。

## 2. 起前置节点（各开一个终端）

```bash
# 相机
ros2 launch orbbec_camera gemini2.launch.py

# UR5 驱动（真实 IP），示教器跑 External Control
ros2 launch ur_robot_driver ur_control.launch.py ur_type:=ur5 robot_ip:=<真实IP>
```
> 也可给标定 launch 加 `enable_camera:=true enable_robot:=true robot_ip:=<IP>` 让它顺带起。

先确认图像和 TF 正常：
```bash
ros2 topic hz /camera/color/image_raw
ros2 run tf2_ros tf2_echo base_link tool0     # 能打印 => 机器人 TF 正常
```

## 3. 启动标定

```bash
ros2 launch bin_picking_bringup calibrate_handeye.launch.py \
    marker_id:=26 marker_size:=0.06
```
- 若相机彩色话题名不同，改 `image_topic:=... camera_info_topic:=...`
  （用 `ros2 topic list | grep color` 查实际名）。
- 确认能检测到码：`ros2 run tf2_ros tf2_echo camera_link handeye_marker`
  应随标定板移动而变化；`ros2 topic echo /aruco_single/pose` 有输出。

## 4. 采样求解（easy_handeye2 GUI）

1. GUI 里会出现采样面板。**手动/低速**移动机器人到一个能清晰看到码的姿态，
   点 **Take Sample**。
2. 换姿态（尽量让码有大角度变化、覆盖视野不同位置），共采 **15~20 组**。
   姿态越多样、覆盖越广，外参越准。
3. 点 **Compute**，看解算结果与残差。
4. 点 **Save** —— 结果存到 `~/.ros2/easy_handeye2/handeye_ur5_gemini2.calib`。

## 5. 写入本项目配置

```bash
python3 scripts/handeye_to_yaml.py            # 默认读上面的 .calib
# 或： python3 scripts/handeye_to_yaml.py --name handeye_ur5_gemini2
```
它会覆盖 `bin_picking_bringup/config/handeye_result.yaml`。然后重编让静态 TF 生效：
```bash
cd ros2_ws && colcon build --packages-select bin_picking_bringup
source install/setup.bash
```

## 6. 精度验证（关键，安全无自动运动）

起系统（会用新外参发 `base_link→camera_link` 静态 TF）+ 验证节点：
```bash
ros2 launch bin_picking_bringup system.launch.py \
    enable_grasp:=false enable_perception:=false robot_ip:=<IP>
ros2 run bin_picking_bringup handeye_verify
```
在 **RViz2**：
1. Fixed Frame 设 `base_link`，Add PointCloud2（相机点云）、Marker(`/handeye_verify/marker`)。
2. 用工具栏 **Publish Point** 点场景里一个**已知物理点**（标定板角、桌上尖点）。
3. 终端会打印该点在 `base_link` 下的坐标，RViz 出现一个红球。
4. 在 MotionPlanning 面板把目标设成 `/handeye_verify/target_pose`（夹爪竖直向下），
   **低速、手握急停** Plan + Execute，让 TCP 去那个点。
5. **量 TCP 实际位置与真实物理点的偏差，应在毫米级。**

偏差大 → 标定不准：检查 `marker_size` 是否量错、采样姿态是否太少/太集中、
码是否翘边反光，然后重做第 3~5 步。

## 7. 常见问题

| 现象 | 排查 |
|---|---|
| 检测不到码 | 光照/反光；`marker_id`、字典是否一致；码是否在视野且不糊 |
| `tf2_echo camera_link handeye_marker` 无输出 | aruco 没检测到；image/camera_info 话题名不对 |
| 外参有恒定偏移 | `marker_size` 与实物不符（最常见）；相机内参未标定好 |
| 结果每次差很多 | 采样太少或姿态太集中；码固定不牢 |
| verify 报 TF 转换失败 | 手眼静态 TF 没发（先跑 system.launch），或点的 frame 不对 |

完成后回到 `docs/03_测试步骤.md` 的测试 5（感知）继续。
