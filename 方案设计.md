# UR5 无序抓取（Bin Picking）项目 — 方案设计（ROS2 版）

> 目标：UR5 机械臂从料框中抓取**无序混摆的零件**，放到指定位置（含高精度定向装配）。
> 相机：奥比中光 Gemini2（RGB-D）。夹爪：二指平行夹爪。零件：自行 3D 打印（**有 CAD 模型**）。
> 框架：**Ubuntu 22.04 + ROS2 Humble**。

---

## 一、总体策略：分三阶段递进

| 阶段 | 内容 | 技术 | 依赖 |
|---|---|---|---|
| **阶段一（当前）** | 传统方法打通闭环 | PPF + ICP 位姿估计 + MoveIt2 | 无需 GPU / 数据 |
| **阶段二** | 大模型增强 | SAM2 + FoundationPose + AnyGrasp | GPU |
| **阶段三** | 具身智能 | Diffusion Policy / VLA(π0) | GPU + 遥操作数据 |

**核心思想**：阶段一把"相机→标定→感知→规划→抓取→放置"整条 ROS2 链路跑通；后续阶段**只替换感知/抓取节点内部算法**，外围节点（驱动、TF、MoveIt2、标定、放置补偿）全程复用。

---

## 二、硬件与布局

- **机械臂**：UR5（`ur_robot_driver` + ros2_control）
- **相机**：Gemini2，**Eye-to-Hand**（固定于料框正上方俯视），`OrbbecSDK_ROS2`
- **夹爪**：二指平行夹爪
- **零件**：3D 打印件，设计时**避免完全对称、带可区分特征**（利于位姿估计消歧）

## 三、ROS2 节点架构

```
┌────────────────┐  /camera/depth/color/points  ┌──────────────────┐
│ orbbec_camera  │ ───────────────────────────→ │ perception_node  │
│ (官方驱动节点) │                              │ (PPF+ICP 位姿估计)│
└────────────────┘                              └────────┬─────────┘
                                                         │ /detected_pose (PoseStamped)
                                                         ↓
        ┌──────────┐      TF2 坐标树         ┌────────────────────┐
        │  TF2     │ ←───────────────────→  │ grasp_planner_node │
        │ (坐标管理)│  相机/基座/工具/工件    │ (抓取位姿+碰撞场景) │
        └──────────┘                        └─────────┬──────────┘
                                                      │ 目标位姿
                                                      ↓
        ┌──────────────┐   关节轨迹    ┌────────────────────────┐
        │ ur_robot_    │ ←──────────  │       MoveIt2          │
        │ driver       │              │ (规划 + 避障 + 执行)    │
        └──────────────┘              └────────────────────────┘
                  ↑                                  ↑
            真实 UR5                          RViz2（可视化调试）
```

## 四、阶段一：传统方法（详细方案）

### 4.1 系统流水线
```
Gemini2 采集 RGB-D / 点云 (OrbbecSDK_ROS2)
      ↓
点云预处理（去背景/料框、降采样、平面分割、聚类） — Open3D/PCL
      ↓
6D 位姿估计（CAD ↔ 场景点云：PPF 粗配准 → ICP 精配准）→ PoseStamped
      ↓
TF2 坐标变换（相机系 → 机器人基座系）
      ↓
抓取位姿生成（CAD 预标注抓取点 + 当前位姿）+ 料框碰撞模型加入 Planning Scene
      ↓
MoveIt2 规划无碰撞轨迹：接近 → 抓取 → 抬起
      ↓
在手位姿重估计 + 几何补偿 → MoveIt2 精确定向放置 / 装配
```

### 4.2 软件栈
| 模块 | 工具 |
|---|---|
| 相机驱动 | `OrbbecSDK_ROS2`（官方） |
| 点云处理 | `Open3D` / `PCL`（节点内调用） |
| 6D 位姿估计 | `opencv-contrib`（PPF）+ Open3D（ICP） |
| 坐标管理 | **TF2** |
| 手眼标定 | **easy_handeye2**（ChArUco，Eye-to-Hand 模式） |
| 运动规划/避障 | **MoveIt2** |
| 机械臂驱动 | `ur_robot_driver` + ros2_control |
| 可视化 | **RViz2** |
| 状态机 | 简单状态机 / `behaviortree_cpp` |

### 4.3 开发步骤（路线图）
- **阶段 0｜环境搭建**：装 Ubuntu 22.04 + ROS2 Humble；安装 `ur_robot_driver`、`OrbbecSDK_ROS2`、`moveit2`、`easy_handeye2`。
  - 验证：`ros2 launch` 起相机看到点云话题；起 UR 驱动，RViz2 中机械臂随实物联动。
- **阶段 1｜手眼标定**：`easy_handeye2` 走 Eye-to-Hand 流程 → 自动写入 TF 树。
  - 验证：相机看到一点，机器人能准确走过去。
- **阶段 2｜感知节点**：订阅点云 → 预处理 → PPF+ICP → 发布 `/detected_pose`。
  - 验证：RViz2 中 CAD 模型贴合到场景点云。
- **阶段 3｜抓取规划 + 避障**：CAD 预标注抓取点；料框/支架加入 MoveIt **Planning Scene**；MoveIt2 规划接近→抓取→抬起无碰撞轨迹并执行。
  - 验证：从料框稳定抓起零件且不撞料框。
- **阶段 4｜放置**：在手位姿重估计 + 几何补偿；MoveIt2 规划到放置位姿，TF2 自动算工件↔放置点变换。
  - 验证：装配精度达标。
- **阶段 5｜状态机/鲁棒性**：整体循环 + 抓取失败检测、重试、料框清空逻辑。

### 4.4 关键技巧
- **MoveIt2 碰撞场景**：把料框、相机支架建成碰撞体，这是 ROS2 方案最大价值（伸进料框抓取的避障）。
- **在手位姿重估计**：抓起后举到相机前，用 CAD 做位姿估计得到 `T_gripper→part`，反推精确放置位姿 —— 高精度装配核心。

## 五、阶段二：大模型增强（后续）

在阶段一基础上，**只替换感知/抓取节点内部算法**，ROS2 架构不变：

| 位置 | 模型 | 作用 | 备注 |
|---|---|---|---|
| 分割检测 | SAM2 / Grounding-DINO | 开放词汇分割零件 | 封装为节点 |
| 6D 位姿 | **FoundationPose** | 无训练位姿估计，替代/增强 PPF | 需 CAD + mask |
| 抓取生成 | **AnyGrasp** | 预训练抓取检测，免采数据 | 需申请 license |

**推荐组合（免自训练）**：SAM2 找零件 → AnyGrasp 抓取 → FoundationPose 在手位姿 → 几何补偿精放。
（已在 Ubuntu 上，无需迁移；注意各模型显存预算。）
