## 1. 项目是什么
UR5 机械臂 + 奥比中光 Gemini2 相机，从料框抓取**无序混摆的 3D 打印零件**，
放到指定位置（含**高精度定向装配**）。

## 2. 关键决策（已敲定）
- **零件**：用户自行 3D 打印 → **有 CAD 模型** → 走"基于模型的位姿估计"，非无模型抓取。
- **框架**：Ubuntu 22.04 + **ROS2 Humble**。组件：ur_robot_driver / OrbbecSDK_ROS2 / MoveIt2 / easy_handeye2。
- **相机布局**：Eye-to-Hand（固定俯视料框）。
- **夹爪**：沃姆 **EPGC-50-150-B-C-NM-L2**（行程 50mm、负载 3kg）。
  接线：供电=UR 工具口 24V(8针航插，**控制柜=CB3** 限流600mA，实际夹持电流待真机读 0x1117 核对)，
  通讯=飞线 485A/B→USB转RS485 转换器→主机 `/dev/ttyUSB0`（Modbus-RTU）。
  手指为用户自制二指（width_offset 待标定）。波特率/站号出厂默认待真机扫描确认。
  ⚠️ 手册寄存器表页眉为"EPG2系列"（协议通用），型号相关量(行程等)驱动会从 0x1114 读实际值。
- **UR5 IP**：`192.168.0.11`（真机实测）。启动需加 `rtde_config_package_timeout:=500`(ms)，否则网络延迟致 RTDE 握手超时、`ur_ros2_control_node` SIGABRT。
- **路线分三阶段**：①传统 PPF+ICP（当前已实现）→ ②大模型(SAM2/FoundationPose/AnyGrasp) → ③具身(Diffusion Policy/π0)。
- **运动规划**：自由移动 RRTConnect + 抓取接近/抬起用笛卡尔直线 + 料框碰撞避障。
- **高精度装配**：在手位姿重估计 + 放置补偿（混合方案：学习/几何分工）。
- **开发工作流**：在 **Windows**(`E:\zhuomian\robotarm`) 写代码 → 拷到**另一台 Ubuntu 双系统机**构建测试。

### 节点清单与数据流
```
pick_loop ──/pick_place/run──> grasp_executor ──MoveIt2/笛卡尔──> UR5
perception_node ─/detected_objects─> grasp_planner ─/grasp_candidates─> (pick_loop & executor)
grasp_executor ──/gripper/set_gripper──> gripper_driver(沃姆USB)
grasp_executor ──/estimate_inhand──> inhand_estimator (阶段4)
handeye 静态TF(base<-camera) 由 system.launch 发布
```
可执行节点：`perception_node`, `gripper_driver`, `grasp_planner`, `grasp_executor`,
`inhand_estimator`, `pick_loop`, `publish_test_grasp`, `handeye_verify`。
阶段二新增（`bin_picking_perception_v2`）：`sam6d_client`(→/detected_objects),
`foundationpose_client`(→/detected_objects_inhand, 服务 /inhand_pose/reset),
`grasp_client`(→/grasp_candidates)；三者经 ZMQ 调 `inference/server.py`。
